# uCOS Car Speed Estimate

uC/OS-II 기반 자동차 보조장치 시뮬레이터

RTOS 환경에서 입력 처리, 상태 계산, 이벤트 감지, UI 출력을 독립 태스크로 분리해 구현한 개인 프로젝트입니다. 자동차 속도와 RPM을 단순 모델로 계산하고, 급발진처럼 보이는 상황을 감지했을 때 긴급 제동과 경고 메시지를 수행하도록 구성했습니다.

## 프로젝트 목적

이 프로젝트의 목표는 단순히 차량 속도를 계산하는 것이 아니라, RTOS 기반 시스템에서 기능을 어떻게 태스크 단위로 분리하고 서로 어떻게 협력하게 만들지 직접 구현해보는 것이었습니다.

특히 아래 세 가지를 중점적으로 다뤘습니다.

- 입력, 계산, 모니터링 기능을 독립 태스크로 분리하는 구조 설계
- 공유 상태를 다루기 위한 동기화와 이벤트 전달 방식 구현
- 이상 상황을 감지하고 즉시 대응하는 안전 기능 시뮬레이션

## 핵심 구현 포인트

### 1. 기능을 RTOS 태스크로 분리

하나의 루프에서 모든 기능을 처리하지 않고, 역할별로 태스크를 분리했습니다.

- `inputhandler()`: 사용자 입력 처리
- `SUAhandler()`: 급발진 시나리오 주입
- `rpmhandler()`: RPM 계산
- `gearhandler()`: 기어 상태 갱신 및 속도 계산
- `speedhandler()`: 브레이크 반영 및 RPM 동기화
- `eventhandler()`: 급발진 패턴 감지
- `monitorPrinter()`: 현재 상태 출력

이 구조를 통해 각 기능의 책임을 분리했고, RTOS 기반 설계에서 중요한 동시성 사고방식을 직접 적용했습니다.

### 2. 공유 상태와 이벤트 전달을 분리

시스템은 `speed`, `rpm`, `break_pad`, `accel_pad`, `gear_in`, `gear_state` 같은 전역 상태를 중심으로 동작합니다.

- 입력 관련 상태는 `inputSem` 세마포어로 보호
- 경고 이벤트는 `msgQueue` 메시지 큐를 통해 출력 태스크로 전달

단순히 값을 바꾸는 수준을 넘어서, 입력 보호와 이벤트 전달 방식을 서로 다르게 설계해 RTOS에서 자주 사용하는 통신 패턴을 경험했습니다.

### 3. 급발진 감지와 긴급 제동 로직 구현

`eventhandler()`는 아래 조건이 3주기 연속 발생하면 급발진으로 판단합니다.

- 가속 페달 입력이 0
- 속도가 이전 주기보다 증가

조건이 충족되면:

- `eventWriter()`가 `break_pad = 9`로 설정해 즉시 최대 제동
- 메시지 큐를 통해 경고 전달
- 모니터 화면에 `Emergency break opened!` 출력

이 부분은 단순 계산 프로젝트에서 끝나지 않고, 이상 징후 감지와 대응까지 포함한 제어 흐름을 구현했다는 점에서 의미가 있습니다.

### 4. 차량 상태 계산을 단계별로 모델링

속도와 RPM은 아래 흐름으로 계산됩니다.

1. 가속 페달 입력을 받아 RPM 증가
2. 자연 감속 로직으로 RPM 감소
3. 기어비를 이용해 속도 계산
4. 브레이크 입력으로 속도 감소
5. 변경된 속도에 맞춰 RPM 재동기화

실제 차량 물리를 정밀하게 구현한 것은 아니지만, 상태 변화가 연속적으로 이어지도록 모델링해 시뮬레이터로서 충분한 일관성을 갖추도록 했습니다.

## 시스템 흐름

```text
Keyboard Input
    -> inputhandler
    -> shared state update
    -> rpmhandler / gearhandler / speedhandler
    -> eventhandler
    -> message queue
    -> monitorPrinter
```

상태 계산과 UI 출력이 분리되어 있기 때문에, 계산 로직 변경이 화면 출력 구조를 직접 흔들지 않도록 설계했습니다.

## 실행 방법

빌드를 위해서는 아래 환경이 필요합니다.

- `C:\SOFTWARE\uCOS-II\SOURCE`
- `C:\SOFTWARE\uCOS-II\Ports\80x86\WIN32\VC\src`
- Microsoft Visual C++ 컴파일러 `cl`
- `winmm.lib`, `user32.lib`

`source/makefile` 기준 빌드 명령은 아래와 같습니다.

```make
all:
	@cl -nologo /MD /I$(UCOS_SRC) /I$(UCOS_PORT_SRC) /I$(UCOS_PORT_EX) $(EXAMPLE) $(UCOS_SRC)\ucos_ii.c $(UCOS_PORT_SRC)\pc.c $(UCOS_PORT_SRC)\os_cpu_c.c winmm.lib user32.lib
```

실행 중 조작 키:

- `R`, `F`: 브레이크 증가, 감소
- `T`, `G`: 가속 증가, 감소
- `7`, `Y`, `H`, `N`: `P`, `R`, `N`, `D` 기어 전환
- `J`, `M`: 수동 모드 기어 증가, 감소
- `1`, `2`: 급발진 시나리오 시작, 종료
- `Esc`: 종료

## 마무리

포트폴리오 관점에서는 다음을 가장 잘 보여주는 프로젝트라고 생각합니다.

- 태스크 기반 시스템 설계 경험
- 동기화와 이벤트 전달에 대한 이해
- 상태 기반 제어 로직 구현 경험
- 학습 내용을 코드 구조로 정리해내는 능력
